Infrastructure Notes

Deployment Target

Single VPS (4 vCPU, 8GB RAM) is sufficient for 100 concurrent users. Docker Compose manages Next.js, NestJS, PostgreSQL, Redis. Easily upgradeable to separate servers as you scale.

Redis Caching Strategy

Cache course catalog (TTL 10min), user session tokens, JWT blacklist for logout, and BullMQ job queues. Estimated hit rate ~70% for read-heavy LMS operations like course listing and content viewing.

Quick Prototype Path

Start with docker-compose.yml on a single host. Use Prisma migrations for schema from drawio. GitHub Actions → SSH deploy. Estimated time to working prototype: 1–2 weeks.

Option 2

AWS Serverless is powerful for scale but adds significant complexity for a prototype. Cold starts and Lambda function limits can hurt real-time LMS experiences. Better suited for a v2 migration once the product is validated.

Infrastructure Notes

Serverless Trade-offs

Lambda cold starts (100–500ms) impact quiz/assessment user experience. Aurora Serverless v2 has connection pool limits — use RDS Proxy. IaC via AWS CDK or Terraform recommended from day one.

Cost at Prototype Scale

For 100 concurrent users, AWS serverless may cost more than a VPS. Aurora Serverless minimum cost ~$50–70/mo even with low traffic. Lambda + API Gateway costs scale well but add up with RDS Proxy.

When to Choose This

Choose AWS serverless if you expect rapid scale beyond 500+ concurrent users post-prototype, have a team experienced with AWS, or have compliance requirements that mandate cloud-managed infrastructure.

Verdict

★ Option 1 — NestJS + Next.js + Redis

The clear winner for your prototype. Your 100 CCU target is well within a single VPS capacity. NestJS modules map directly to your DB schema groups (auth, courses, students, assessments). Redis handles session management, JWT blacklisting, and course catalog caching out of the box. You'll be deploying in days not weeks.

Fast Prototype Low Cost Full Control Easy Debug Scalable to ~500 CCU

Option 2 — AWS Serverless

Powerful for production scale, but over-engineered for your current phase. Adds 3–4x development complexity, higher monthly costs, and significant AWS-specific knowledge requirements. Recommend migrating to this architecture at v2 or when you consistently exceed 500+ concurrent users.

Pipeline Jobs Detail

PR Checks (all branches)

ESLint + Prettier checks. Unit tests via Jest (NestJS). TypeScript compile check. Prisma schema validation. Block merge on failure.

Build (main/dev)

Docker multi-stage build for NestJS API and Next.js frontend. Push images to GitHub Container Registry (GHCR). Tag with git SHA for rollback.

DB Migrations

Run prisma migrate deploy via SSH before container restart. Zero-downtime with Prisma's migration system. Never run on PRs — only on merge to main.

Rollback Strategy

Keep previous 3 image tags in GHCR. One-command rollback: docker-compose pull [tag] && docker-compose up -d. DB rollback via Prisma migrate resolve.

Environment Strategy

🧪 Development

Local docker-compose with hot reload. Seeded test data. All services on localhost. .env.local for secrets. No CI required.

🔬 Staging

Auto-deploy from dev branch. Full stack on separate subdomain. Production-like config. Used for QA and demo. Deployed after PR merge to dev.

🚀 Production

Deploy on merge to main only. Requires passing all CI checks. GitHub Secrets for env vars. Nginx + TLS. Zero-downtime container swap.
